## 🏗️ GATEWAY MVP ARCHITECTURE (Days 6-8)

### Overview

During Quiet Operations Days 6-8 (2026-05-13 to 2026-05-15), we built a production-ready API gateway from scratch to replace the orphaned `app/main.py` code from dangling commit 99c5e61.

### Architecture Decisions

**Decision 1: Build from Scratch vs. Recover Orphaned Code**
- **Choice:** Build `gateway_mvp.py` from scratch instead of merging commit 99c5e61
- **Rationale:** 
  - Orphaned code was v8.0 draft, not tested or reviewed
  - Simpler implementation easier to maintain and audit
  - Avoids merging unreviewed code into frozen v7.5 architecture
- **Outcome:** ✅ Cleaner codebase, faster development (2 days vs. estimated 1 week)

**Decision 2: PostgreSQL Events Table vs. Neo4j Event Bus**
- **Choice:** Use existing PostgreSQL `events` table with hash chain triggers
- **Rationale:**
  - Hash chain infrastructure already in place (compute_event_hash, validate_hash_chain triggers)
  - No additional dependencies (Redis/RabbitMQ) required
  - Simpler deployment and maintenance
- **Outcome:** ✅ 420+ immutable events logged with cryptographic integrity

**Decision 3: In-Memory Rate Limiting vs. Distributed Store**
- **Choice:** In-memory sliding window algorithm
- **Rationale:**
  - Single-instance gateway (no horizontal scaling yet)
  - Faster implementation (30 min vs. 2-3 hours for Redis)
  - Sufficient for current traffic levels
- **Outcome:** ✅ Rate limiting working perfectly (429 at #11 request)

**Decision 4: Auto-Restart Script vs. Process Manager**
- **Choice:** Simple batch script (`start_gateway.bat`) with loop
- **Rationale:**
  - No external dependencies (no systemd/supervisor on Windows)
  - Immediate crash recovery (5-second restart delay)
  - Operator-visible (can see restart messages)
- **Outcome:** ✅ Gateway recovers automatically from crashes

### Lessons Learned

1. **Gateway Crashes Are Inevitable**
   - Problem: Gateway died during W11 testing (process crash)
   - Impact: All API requests failed with connection refused
   - Solution: Auto-restart script prevents manual intervention
   - Lesson: Production systems need automatic recovery mechanisms

2. **Hash Chain Triggers Only Validate, Don't Compute**
   - Discovery: PostgreSQL triggers RAISE EXCEPTION if hash is NULL
   - Implication: Application must compute hash before INSERT
   - Implementation: `emit_event()` computes SHA-256(record_json + previous_hash)
   - Lesson: Database constraints enforce integrity but don't provide it

3. **Rate Limiting Must Be Per-Endpoint Category**
   - Problem: Different endpoints have different abuse potentials
   - Solution: Categorize endpoints (public, gdpr, complaints)
   - Configuration: Public 40/min, GDPR 5/hr, Complaints 10/hr
   - Lesson: One-size-fits-all rate limiting is insufficient

4. **W11 Enforcement Works at API Level**
   - Test: Created SILENT_DEATH.flag → health returns DEGRADED
   - Result: GDPR erasure blocked with "System DEGRADED" error
   - Recovery: Removed flag → system returned to HEALTHY
   - Lesson: File-based flags provide simple, effective governance
