## ⚠️ KNOWN LIMITATIONS (v7.5)

These are **ACCEPTED** limitations for v7.5, to be addressed in v8.0:

### 1. No Log Rotation
- Current: Unlimited growth (~2 KB per operation)
- Impact: Storage concerns at scale
- Mitigation: Manual cleanup or external rotation
- v8.0 Fix: `core/log_rotation.py` with retention policy

### 2. No Session Context
- Current: Individual operation logs only
- Impact: Limited workflow continuity tracking
- Mitigation: Correlation IDs provide basic linking
- v8.0 Fix: Session-aware logging with workflow tracking

### 3. No Emergency Stop
- Current: No global runtime lock mechanism
- Impact: Cannot halt all CLI operations centrally
- Mitigation: Manual process termination
- v8.0 Fix: `pos emergency-stop` with multi-sig unlock

### 4. Limited Constitutional Depth in Status
- Current: Basic health metrics only
- Impact: Incomplete governance visibility
- Mitigation: Use `pos flags` for detailed checks
- v8.0 Fix: Enhanced constitutional state reporting
